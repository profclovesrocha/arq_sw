# Arquitetura de Software

![Banner Arquitetura de Software](./banner_arq_sw.png)

## 📝 Visão Geral

Este repositório é dedicado à disciplina de **Arquitetura de Software**, oferecendo um ambiente de aprendizado ativo e prático. Ele foi projetado para guiar estudantes através dos princípios, padrões e práticas essenciais para a construção de sistemas de software escaláveis, seguros e de fácil manutenção. A metodologia empregada foca em **Propósito, Projeto e Desafios**, conectando a teoria à ação real, similar a abordagens como Project-Based Learning (PBL) e Challenge-Based Learning (CBL).

---

## 📂 Estrutura do Repositório

O conteúdo está organizado nas seguintes seções principais:

- **`aulas/`**: Materiais didáticos, slides e ementas das aulas teóricas e práticas.
- **`desafios/`**: Propostas de desafios práticos e projetos, incluindo o desafio final.
- **`cad_alunos/`**: Diretório para organização de informações e participações dos alunos.
- **`DOCUMENTAÇÃO_ DESAFIO 1.pdf`**: Documentação específica para o primeiro grande desafio.
- **`Mind Map_AV1_ARQ_SW.png`**: Mapa mental dos tópicos abordados na Avaliação 1.

---

## 📚 Conteúdo das Aulas

A pasta `aulas/` contém uma série de materiais que cobrem os seguintes tópicos:

- **Fundamentos**: Conceitos básicos, Componentes e Conectores (Aulas 01-03).
- **Padrões**: Configurações e padrões arquiteturais para design de sistemas (Aula 04).
- **Prática com Flask**: Implementação MVC, Blueprints e estruturação profissional (Aula 05).
- **Arquiteturas Avançadas**: Clean Architecture e Hexagonal Architecture (Aula 06).

## 🚀 Desafios e Projetos

O **Seminário Final** propõe o desenvolvimento da arquitetura de um **Sistema de Apoio à Decisão Pedagógica para o IMIP**, focando em tecnologias humanizadas e IA generativa.

### 🏛️ Estrutura do Trabalho Final
1. **Definição Estratégica**: Escolha entre Camadas, Microsserviços ou Eventos.
2. **Design de Componentes**: Módulos de IA, Relatórios e Acolhimento Lúdico.
3. **Implementação Flask**: Organização física com Padrão MVC + Blueprints.
4. **Evolução**: Aplicação de Clean/Hexagonal Architecture (Portas e Adaptadores).
